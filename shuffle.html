How to shuffle an existing deck

How to use the API to shuffle an already created deck

In our previous lesson, we learned how to create a deck with a function using this. Please read these first if you have not already, then come back.

So we have created a deck, but as the response indicates it is not shuffled. Poker requires a shuffled deck, so let's shuffle our deck. This is the API request we can use to do so. Notice how similar it is to the create a new deck request: https://deckofcardsapi.com/api/deck/new/

Notice that the only difference is the deck_id of the deck we want to be shuffled must be passed in the query string. </shuffle/ is also included in the URL instead of /new/. The response format remains the identical as the new deck response, but if it is successful, it should indicate the same deck is now shuffled via a true boolean value.

The following code simply adds onto our previous create a deck function. We create a new function shuffleDeck(deck_id) which accepts our saved deckID variable and uses this parameter to append the apiURL variable to ensure our shuffle request is sent to the correct URL. We call this function, passing deckID and save the APIs parsed JSON response to deckData Take a look, and test the button on the results tab.

As we know from before, our newDeck method returned a new unshuffled deck. This output shows that the deck passed to shuffleDeck is the same as the one we created in createDeck, but the output also demonstrates that shuffleDeck did in fact shuffle our deck, proven by the true value of deckData's shuffled property.


JSFiddle calls our functions each time you run them, therefore, your deck_id property value will not match the one below. But just like the below, you should observe that the deck_id used in both methods are identical.

That's it, simple right? You know now how to 1) create a new, unshuffled deck (and simply doing so via a custom method), and 2) shuffle that deck (also with a simple function call). You can now continue to shuffle the same deck over and over as needed, something we will need to do once we learn how to draw cards from a deck.
